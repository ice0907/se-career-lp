### B-1【スレッド：なぜ今、SEに提案力が必要なのか】

**1投目（フック）**
```
AIがコードを書く今、
技術力だけのSEはどうなるか。

元請け企業で上流SEをやってきた経験から
正直に書きます。

🧵（1/8）
```

**2投目**
```
まず現実として。

McKinseyの試算では
IT業務の約6割がAIで部分的に自動化できるとされている。

「自分の仕事がなくなる」とは言い過ぎだが
「今のポジションのままで単価が上がり続ける」
は相当厳しくなっている。

（2/8）
```

**3投目**
```
一方で、減らない需要がある。

Salesforceの調査によると
ビジネスバイヤーの87%が
担当者に「信頼できるアドバイザー」として
振る舞うことを期待している。

課題を理解して、整理して、提案できる人。
これはAIにはまだできない。

（3/8）
```

**4投目**
```
では何が単価を上げるのか。

答えは「Trust Equation（信頼方程式）」にある。

信頼 ＝（信憑性 ＋ 信頼性 ＋ 親密性）÷ 自己志向性

・根拠ある提案ができる（C）
・期日を守り報告できる（R）
・顧客の本音を引き出せる（I）
・顧客課題を自分の利益より優先（÷S）

これが高い人が「代替しにくい人材」になる。

（4/8）
```

**5投目**
```
SEにとって一番得しやすいのがここ。

顧客との会話が業務に組み込まれているので
この力を伸ばすのに
「営業職に転換する」必要はない。

ヒアリング
→ 課題整理
→ 提案
→ 定例会での関係構築

これを現場でやるだけで
顧客から「この人は外せない」と思われやすくなる。

（5/8）
```

**6投目**
```
単価はどう変わるか。

現場で役割が広がった実績を持って
更新交渉をすると通りやすくなる。

具体的には

「当初の開発支援に加え、
　要件整理・顧客調整まで担っています」

これが言えると
単価5〜20万/月の見直しが交渉できる。

（6/8）
```

**7投目**
```
気をつけたいのは

「営業をやれ」と言われると嫌がる人が多いこと。

でも「上流で価値を出せる提案型SEになる」
と言い換えると、同じ内容でも前向きになれる。

言葉の問題だけじゃなく、本質が違う。

売り込みじゃなく、顧客の課題を解く力。
これを身につけた人が次の時代で残る。

（7/8）

```

────────────────

「コードを書く・設計する」という仕事は

これは脅しじゃなく、実態だ。

では、何が残るのか。

答えは「顧客との信頼」だ。

────────────────

顧客が本当に困っているのは
「技術的に正しいシステムがない」ことではない。

「自分たちの課題を理解してくれる人がいない」ことだ。

でも顧客の本音を引き出して、整理して、
「こうしませんか」と言える人は代替できない。

────────────────

その力は3つに分解できる。

① ヒアリング力
「何を作りますか？」ではなく
「何に一番困っていますか？」が言える力。

要件の背景にある本当の課題を掘り起こす。

引き出した課題を構造化して

③ 定例会での信頼構築力

「この人は分かってくれる」という実感を
積み上げる場だ。

顧客が「ちょっと相談していいですか」と
言ってくれるようになれば、
単価交渉は自然に通りやすくなる。

────────────────

これは営業力ではない。

テレアポでも飛び込みでもない。

顧客の仕事を前に進める力だ。

この力を持つSEは
